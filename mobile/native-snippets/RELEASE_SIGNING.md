# Release signing & Play Store readiness

The `mobile/` kit is source-only; the native `android/` project is generated by the RN CLI
(see `../SETUP_ANDROID.md`). These are the changes to make that generated app **production-
signed and Play-Store-ready**. Apply them to `android/` after generating and copying the kit in.

> The debug build ships with the shared Android debug keystore. A Play Store release **must**
> be signed with your own private upload key. You (a human) create the keystore and hold the
> passwords — they can't live in the repo.

---

## 1. Create the upload keystore (once)

Run in `android/app/` (needs JDK 17's `keytool` on PATH):

```bash
keytool -genkeypair -v \
  -storetype PKCS12 \
  -keystore kokonada-upload.keystore \
  -alias kokonada-upload \
  -keyalg RSA -keysize 2048 -validity 10000
```

Answer the prompts and choose strong store/key passwords. **Back this file up** — losing it
means you can never ship an update under the same app (unless enrolled in Play App Signing,
which is recommended: Play holds the app-signing key and you rotate only the upload key).

## 2. Keep secrets out of git

Copy `keystore.properties.example` → `android/keystore.properties`, fill in the real values,
and add both of these to `android/.gitignore`:

```
keystore.properties
app/*.keystore
```

## 3. Wire signing into `android/app/build.gradle`

At the **top** of the file (above `android { }`):

```gradle
def keystorePropertiesFile = rootProject.file("keystore.properties")
def keystoreProperties = new Properties()
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}
```

Inside `android { }`, replace the default `signingConfigs`/`buildTypes.release` with:

```gradle
    signingConfigs {
        release {
            if (keystorePropertiesFile.exists()) {
                storeFile file(keystoreProperties['storeFile'])
                storePassword keystoreProperties['storePassword']
                keyAlias keystoreProperties['keyAlias']
                keyPassword keystoreProperties['keyPassword']
            }
        }
    }
    buildTypes {
        release {
            // Real release signing (NOT the debug key).
            signingConfig signingConfigs.release
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile("proguard-android-optimize.txt"), "proguard-rules.pro"
        }
    }
```

## 4. App identity & versioning (in `android/app/build.gradle` → `defaultConfig`)

```gradle
        applicationId "com.kokonada.health"   // final, reverse-DNS; can't change post-launch
        minSdkVersion 26
        targetSdkVersion 35                    // Play requires a recent target each year
        versionCode 1                          // integer — bump +1 on EVERY upload
        versionName "1.0.0"                     // human-facing semver
```

**Versioning rule:** `versionCode` must strictly increase for each AAB you upload; bump
`versionName` for user-visible releases.

## 5. ProGuard/R8

Append the rules in `proguard-rules.additions.pro` to `android/app/proguard-rules.pro`
(keeps react-native-ble-plx, react-native-health-connect, and RN reflection intact under R8).

## 6. Adaptive launcher icon

Replace the generator defaults with a real adaptive icon:
`android/app/src/main/res/mipmap-*/ic_launcher.*` and `ic_launcher_round.*`, plus
`mipmap-anydpi-v26/ic_launcher.xml` (foreground + background layers). Android Studio's
**Image Asset** wizard (New → Image Asset → Launcher Icons) generates every density.

## 7. Build the release bundle

```bash
cd android
./gradlew bundleRelease        # → app/build/outputs/bundle/release/app-release.aab
```

Verify it's signed with your key (not the debug key):

```bash
jarsigner -verify -verbose -certs app/build/outputs/bundle/release/app-release.aab | head
```

## 8. Play Console checklist (manual, done by a human)

- Create the app in the Play Console; set `applicationId` to match `com.kokonada.health`.
- Enrol in **Play App Signing** (recommended).
- Complete: Privacy Policy URL, Data safety form (declare **health/biometric** data:
  heart rate, HRV, sleep — read-only, used to personalise playlists, deletable in-app via
  the account-deletion flow), Health apps declaration, content rating, target audience.
- Upload `app-release.aab` to the Internal testing track first; promote after QA.

> **What this repo can't do for you:** generate your private keystore, hold your passwords,
> or upload to the Play Console — those require your Google account and secrets. Everything
> up to `bundleRelease` is scripted above.
