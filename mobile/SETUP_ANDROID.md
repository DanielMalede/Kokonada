# Kokonada Health — Android companion (Health Connect) — Setup on Windows

Android-only MVP. Reads Garmin-synced data from **Google Health Connect** and pushes it to
`POST /api/integrations/health/batch` (already built on the backend). No iOS, no Mac needed.

> **This `mobile/` folder is a drop-in kit**, not a full React Native project (the native
> `android/` project is generated by the RN CLI). You generate a fresh RN app, then copy
> `mobile/src/` into it and apply the native snippets in `mobile/native-snippets/`.

---

## 0. Prerequisites (Windows)

| Tool | Notes |
|---|---|
| **Node.js LTS** (≥ 18) | `node -v` |
| **JDK 17** (Temurin/Adoptium) | RN 0.74+ needs JDK 17. Set `JAVA_HOME`. |
| **Android Studio** | Install an **SDK Platform** (API 35 or 36), **Platform-Tools**, **Build-Tools**. |
| **Env vars** | `ANDROID_HOME=%LOCALAPPDATA%\Android\Sdk`; add `%ANDROID_HOME%\platform-tools` to `PATH`. |
| **Physical device** | Enable **Developer options → USB debugging**, plug in USB, accept the RSA prompt. |
| **Health Connect** | On Android 14+ it's built into the OS. On Android 13 and below, install it from Play Store (`com.google.android.apps.healthdata`). **The user must also enable Garmin Connect → Settings → Health Connect → turn on sharing** — this is the unavoidable prerequisite. |

Verify the device is visible:
```bash
adb devices          # should list your device as "device" (not "unauthorized")
```

## 1. Generate the React Native project

```bash
# from the repo root (creates a sibling RN project; keeps the kit separate)
npx @react-native-community/cli@latest init KokonadaHealth
cd KokonadaHealth
```

## 2. Install packages

```bash
npm install react-native-health-connect          # Health Connect
npm install @react-native-google-signin/google-signin   # auth (Google Sign-In)
npm install react-native-keychain                # secure JWT storage
npm install react-native-ble-plx                 # live heart rate over BLE
```

> **BLE (react-native-ble-plx):** after `npm install`, the library is autolinked. It needs
> the Bluetooth permissions in `native-snippets/AndroidManifest.additions.xml` (already
> included) and a minSdk of 23+ (this kit targets 26). On the device, the user enables
> **Garmin watch → Settings → Sensors & Accessories → Broadcast Heart Rate** so the watch
> advertises the standard BLE Heart Rate Service (0x180D) the app subscribes to.

## 3. Copy the kit source in

Copy this kit's `mobile/src/` into the generated project's `src/`:
```bash
# from KokonadaHealth/
xcopy /E /I ..\mobile\src src
```
Then edit `src/health/config.ts`:
- `BACKEND_URL` → your Railway backend (https://…)
- `GOOGLE_WEB_CLIENT_ID` → the OAuth **Web** client ID from step 4e

Auth is fully wired in-app (Google Sign-In → `POST /api/auth/google` → JWT in Keychain →
attached to every `/health/batch` upload). No manual token pasting.

## 4. Native config (apply the snippets in `mobile/native-snippets/`)

### 4a. SDK levels — `android/build.gradle`
Health Connect requires **minSdkVersion 26**. In the top-level `android/build.gradle`,
`buildscript { ext { … } }`:
```gradle
buildscript {
    ext {
        minSdkVersion = 26          // Health Connect requires Android 8.0+
        compileSdkVersion = 36      // matches the API 36 platform already installed
        targetSdkVersion = 36
        // ...leave buildToolsVersion / ndkVersion as generated
    }
}
```

### 4b. Register the permission delegate — `android/app/src/main/java/com/kokonadahealth/MainActivity.kt`
Apply `native-snippets/MainActivity.kt.diff` (adds the `HealthConnectPermissionDelegate`
in `onCreate` — **without this the permission dialog never returns a result**).

### 4c. Privacy-policy rationale screen
Create `android/app/src/main/java/com/kokonadahealth/PermissionsRationaleActivity.kt`
from `native-snippets/PermissionsRationaleActivity.kt` (a WebView showing your privacy policy;
Health Connect deep-links to it).

### 4d. `AndroidManifest.xml`
Merge `native-snippets/AndroidManifest.additions.xml` into
`android/app/src/main/AndroidManifest.xml` — it adds the health `<uses-permission>` entries
(the scope-minimized set: HR / HRV / sleep / resting HR, plus **`READ_HEALTH_DATA_HISTORY`**
for >30-day reads; **no** SpO2/respiratory/background scopes — they have no reader),
the rationale `intent-filter` on `MainActivity`, the `PermissionsRationaleActivity`, and the
Android-14 `activity-alias`.

### 4e. Google Sign-In (OAuth clients)
Google Sign-In on Android needs **two** OAuth clients in Google Cloud Console
(APIs & Services → Credentials), both under the same project:

1. **Web client ID** — type "Web application". Use its ID for:
   - `GOOGLE_WEB_CLIENT_ID` in `src/health/config.ts`, **and**
   - the backend's `GOOGLE_CLIENT_ID` env var (the audience it verifies). They must match.
2. **Android client ID** — type "Android". Enter your **package name**
   (`applicationId`, e.g. `com.kokonadahealth`) and the app's **SHA-1**. Get the debug SHA-1:
   ```bash
   cd android
   gradlew.bat signingReport      # copy the SHA1 under "Variant: debug"
   ```
   (`@react-native-google-signin/google-signin` does **not** require a Firebase
   `google-services.json` — only the Android OAuth client + SHA-1.)

## 5. Wire the screen into the app
In `App.tsx`, render the kit's screen:
```tsx
import ConnectHealthScreen from './src/components/ConnectHealthScreen';
export default function App() { return <ConnectHealthScreen />; }
```

## 6. Build & run on the connected device

```bash
# Terminal A — Metro bundler
npm start

# Terminal B — build + install + launch on the USB device
npx react-native run-android
```

Or build a debug APK directly and install it:
```bash
cd android
gradlew.bat assembleDebug                 # Windows
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

## 7. On-device flow
1. App opens → checks Health Connect availability (`getSdkStatus`).
2. Tap **Sign in with Google** → exchanged for a Kokonada JWT (`POST /api/auth/google`),
   stored in the Android Keystore via Keychain.
3. Tap **Build my medical profile** → single Health Connect permission sheet (HR, HRV, sleep,
   resting HR, SpO2, respiration + history + background) → **Allow all**.
4. App reads the available history (up to 6 months — **forward-accrue**: Garmin may only have
   ~2 weeks–90 days in the store for a new connection; the rest accrues over time), maps it to
   HR + HRV + per-night sleep stages, and uploads it in chunks to `/api/integrations/health/batch`.
5. Tap **Start live heart rate** → grants the Bluetooth permission → streams real-time HR from
   the Garmin "Broadcast Heart Rate" advertisement (BLE Heart Rate Service `0x180D`) to
   `/api/integrations/watch/hr`. **Deny** Bluetooth and it falls back to a strict 3-minute push
   sourced from Health Connect — same endpoint, same live socket → real-time playlist adaptation.

## 8. Release build & Play Store

For a production, Play-Store-signed build (upload keystore, `signingConfigs.release`, ProGuard,
versioning, adaptive icon, Data-safety declaration for health data, and the `bundleRelease`
steps) follow **`native-snippets/RELEASE_SIGNING.md`**. Copy `native-snippets/keystore.properties.example`
→ `android/keystore.properties` and append `native-snippets/proguard-rules.additions.pro` to
`android/app/proguard-rules.pro`.

## Troubleshooting
- **Permission sheet closes with no result** → `HealthConnectPermissionDelegate.setPermissionDelegate(this)` missing in `MainActivity.onCreate` (step 4b).
- **`getSdkStatus` returns 1/2** → Health Connect not installed / needs update → app deep-links to Play Store.
- **Only ~30 days come back** → `ReadHealthDataHistory` was not granted, OR Garmin simply hasn't synced older data yet (expected for new connections — forward-accrue).
- **Empty data** → confirm Garmin Connect → Health Connect sharing is ON and a sync has run.
- **`adb` shows "unauthorized"** → re-accept the RSA fingerprint prompt on the phone.
- **Google Sign-In throws `DEVELOPER_ERROR`** → the Android OAuth client's **SHA-1 or package name doesn't match** (step 4e), or `GOOGLE_WEB_CLIENT_ID` is not the **Web** client ID.
- **Auth returns 401/`invalid token`** → backend `GOOGLE_CLIENT_ID` ≠ the Web client ID used in the app.
